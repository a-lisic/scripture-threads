# Scripture Threads - Aesthetic Direction

## Core Feel

Scripture Threads should feel modern, clean, calm, and study-first.

The app is not trying to feel like a church bulletin, devotional journal, academic database, or ornate Bible software. It should feel like a focused writing and study workspace: clear enough for daily use, thoughtful enough for deeper study, and quiet enough that the Scripture and notes stay central.

The generated header image is the current visual anchor: soft paper, fine thread lines, muted sage, dark ink, and restrained gold. The rest of the interface should support that image without becoming overly decorative.

## Design Words

- Clean
- Thoughtful
- Warm but still clean
- Structured
- Trustworthy
- Editorial
- Calm
- Personal

## Visual Principles

1. Let structure create beauty.
   Use spacing, hierarchy, dividers, and consistent sections before adding decorative elements.

2. Keep the study surface readable.
   The verse notes, reflections, references, and editable export should feel like the main event.

3. Use restrained color.
   Color should guide attention, separate meaning, and support scanning. It should not become the personality of the whole app.

4. Avoid religious decoration as a default.
   No faux parchment, scrolls, stained glass, ornate crosses, heavy gold, or devotional stock imagery unless a specific future context calls for it.

5. Keep controls compact and practical.
   This should behave like a tool the user returns to often, not a landing page.

## Palette Direction

- Base: soft white and warm paper
- Text: deep ink green/black
- Primary accent: muted sage/deep green
- Link/entity accent: muted blue-slate
- Highlight accent: restrained thread-gold
- Warning/accent note: muted rose only when useful

Avoid a palette that reads mostly brown, orange, purple, or dark slate. Warmth is welcome, but the app should still feel crisp and modern.

## UI Direction

- Left panel: compact setup and metadata
- Main panel: spacious reading/editing workspace
- Tabs: quiet, obvious, not decorative
- Entity display: clean labels in the app, Obsidian wikilinks only in export
- Export/editing: rich-text editing first, markdown generated behind the scenes
- Cards: only for repeated or bounded tools, not every section

## Voice In The Interface

The app should use plain, direct labels:

- Study
- Edit Note
- Entities
- Copy
- Export
- Big Takeaways
- Verse Notes
- Cross-References
- Linked Notes

Avoid labels that feel overly formal, cute, or wordy.
